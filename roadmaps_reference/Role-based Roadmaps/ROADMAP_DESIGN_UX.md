## 7. shadcn/ui no Handoff

> Aplicar as instrucoes do `SHADCN_GUIDE.md` ao desenhar telas que serao implementadas em React/Tailwind.

- [ ] Preferir padroes implementaveis com componentes shadcn/ui existentes
- [ ] Especificar `Dialog`, `Sheet`, `Drawer`, `AlertDialog`, `Popover` ou `HoverCard` conforme o caso
- [ ] Usar `Alert` para mensagens inline
- [ ] Usar `Empty` para estados vazios
- [ ] Usar `Sonner` para toasts
- [ ] Usar `Skeleton` para carregamento
- [ ] Usar `Badge` para status e categorias
- [ ] Prever `AvatarFallback`
- [ ] Prever `DialogTitle`/`SheetTitle` mesmo quando visualmente escondido
- [ ] Definir composicao de `Card` com header, content e footer quando aplicavel
- [ ] Documentar variantes de `Button`, `Input`, `Select`, `Tabs`, `Table` e `Sidebar`

## Stack Recomendada para 2026

| Area | Ferramenta/Pratica |
| ---- | ------------------ |
| Design | Figma |
| Pesquisa | Entrevistas + surveys + analytics |
| Prototipos | Figma prototypes |
| Design system | shadcn/ui como referencia implementavel |
| Acessibilidade | WCAG + testes por teclado |
| Handoff | Component specs + estados + tokens |
| Produto | Metrics tree + opportunity solution tree |
| Documentacao | Markdown + links para Figma |
